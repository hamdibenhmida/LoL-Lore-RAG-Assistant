04 — Pipeline
The 6-Stage RAG Pipeline
1️⃣
Query Ingestion
  • Receive user question
  • Input validation
  • Query preprocessing
2️⃣
Document Retrieval
  • Embed query (384-dim)
  • Cosine similarity search
  • Top-K ranking
3️⃣
Context Construction
  • Sort by relevance
  • Format with source info
  • Validate context length
4️⃣
Prompt Engineering
  • Inject context
  • Apply template
  • Anti-hallucination rules
5️⃣
LLM Generation
  • Call chosen provider
  • Temperature + max tokens
  • Streaming support
6️⃣
Response Validation
  • Clean response text
  • Hallucination check
  • Return answer + sources

07 — Implementation
Key Design Decisions
💾
Persistent Vector Store
  • Avoid expensive re-embedding on every restart
  • Faster startup — load existing ChromaDB collection
  • Data stays local — enhanced privacy
  • Sync-with-disk removes stale chunks automatically
🧩
Modular Architecture
  • Ingestion / Retrieval / Generation are independent
  • Swap any component without touching others
  • Easy unit testing per module
  • Clear responsibility boundaries
⚙️
Configurable via .env
  • No code changes needed to tune parameters
  • CHUNK_SIZE · CHUNK_OVERLAP · RETRIEVER_TOP_K
  • Swap LLM provider per deployment
  • Easy reproducibility and experimentation
🛡️
Explicit Hallucination Prevention
  • Prompt explicitly forbids external knowledge
  • ResponseEvaluator detects insufficient-info answers
  • Source attribution in every response
  • Confidence metrics tracked per query

09 — Limitations
Known Limitations
Technical
  • Cannot answer questions outside provided corpus
  • Semantic search may miss keyword-specific queries
  • Requires well-formed, clean input documents
  • Subject to LLM provider API rate limits
  • May miss complex multi-hop logical relationships
Practical & Scalability
  • Output quality depends on input document quality
  • First run requires initial embedding download (~500MB)
  • ChromaDB grows with the document corpus
  • Optimised for lore-scale corpora (<1GB)
  • Would need distributed search for massive datasets
Context: These limitations are inherent trade-offs of a locally-deployed RAG system designed for lore exploration — they are acceptable for the current scope and addressable in future iterations.
